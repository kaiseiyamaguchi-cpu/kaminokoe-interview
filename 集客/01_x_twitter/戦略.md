## コンテンツカテゴリ

### 1. 面接Tips（メイン・1日2本）
```
例:
- 「面接官が『最後に質問ありますか？』と聞く本当の理由」
- 「志望動機で『御社の〇〇に惹かれて』は危険な理由」
- 「自己PRで『コミュ力』を使うと落ちる確率が上がる」
```
**生成方法**: プロンプトテンプレート + Claude API
**バリエーション**: 業界別、職種別で横展開

### 2. 質問例・回答例（1日1本）
```
例:
- 「【コンサル志望者向け】ケース面接の定番質問5選」
- 「【ガクチカ】サークルネタの正しい使い方」
- 「【逆質問】面接官が『この子いいな』と思う質問3選」
```
**生成方法**: 質問データベースから自動抽出 + 解説生成

### 3. データ・統計（週2本）
```
例:
- 「AI面接練習した人の内定率、練習なしと比べて◯%高い」
- 「面接で落ちる人の共通点TOP5（AIが分析）」
- 「25卒の面接平均回数は◯回。あなたは？」
```
**生成方法**: 自社データ蓄積後に本格化

### 4. ユーザーの声・成功報告（週2本）
```
例:
- 「神の声で練習したら、本番で全く緊張しなかった」
- 「AIの指摘通りに直したら、次の面接で通過した」
```
**生成方法**: 初期はテンプレ、ユーザー増加後は実データ

### 5. エンゲージメント狙い（週2本）
```
例:
- 「面接で一番緊張する瞬間は？」（投票機能）
- 「あなたの志望業界は？」（投票機能）
- 「面接あるある言いたい」
```
**生成方法**: テンプレートから自動

## 自動化構成

### ツール構成
```
[コンテンツDB] → [Claude API] → [投稿生成] → [Buffer/Typefully] → [X投稿]
                                      ↓
                              [スプレッドシート管理]
```

### 実装ステップ

1. **プロンプトテンプレート作成**
   - 各カテゴリ用のプロンプト
   - トンマナ指示
   - 文字数制限（280字以内）

2. **コンテンツDB構築**
   - 面接質問データベース（業界×職種）
   - Tips/ネタのストック
   - 投稿履歴（重複防止）

3. **生成スクリプト**
   - Claude API連携
   - 1週間分を一括生成
   - スプレッドシートに出力

4. **スケジューラー連携**
   - Buffer or Typefully
   - スプレッドシートから自動インポート

## 運用ルール

### やること
- 週1回: 生成コンテンツの軽いレビュー
- 月1回: パフォーマンス分析、プロンプト調整

### やらないこと
- 毎日の手動投稿
- 炎上しそうな話題
- 政治・宗教・差別的内容

### リスク対策
- 投稿前に自動でNGワードチェック
- センシティブな企業名は避ける
- 炎上時は即座に投稿停止できる仕組み
